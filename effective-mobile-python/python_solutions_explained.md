## Задача 1. Проверка строк на изоморфность

### Условие

Нужно проверить, являются ли две строки изоморфными.

Строки `s` и `t` считаются изоморфными, если каждый символ первой строки можно заменить символом второй строки так, чтобы получить вторую строку. При этом:

- одинаковые символы первой строки должны всегда заменяться одним и тем же символом;
- разные символы первой строки не могут заменяться одним и тем же символом;
- порядок символов должен сохраняться.

Пример:

```python
s = 'paper'
t = 'title'
print(is_isomorphic(s, t))
# True
```

### Решение

```python
def is_isomorphic(row_1, row_2):
    if len(row_1) != len(row_2):
        return False

    row_dict = {}
    used_chars = set()

    for char_1, char_2 in zip(row_1, row_2):
        if char_1 not in row_dict:
            if char_2 in used_chars:
                return False
            row_dict[char_1] = char_2
            used_chars.add(char_2)
        else:
            if row_dict[char_1] != char_2:
                return False

    return True


s = 'paper'
t = 'title'
print(is_isomorphic(s, t))
```

### Логика решения

Для каждой буквы из первой строки нужно запомнить, в какую букву второй строки она переводится.

Для этого используется:

- словарь `row_dict`, который хранит соответствия символов первой строки символам второй строки;
- множество `used_chars`, которое контролирует, что один и тот же символ второй строки не используется для разных символов первой строки.

Алгоритм проходит по двум строкам одновременно с помощью `zip()` и проверяет каждую пару символов.

Если символ первой строки встречается впервые, создаётся новое соответствие. Если символ второй строки уже был использован для другого символа, возвращается `False`.

Если символ первой строки уже есть в словаре, алгоритм проверяет, совпадает ли текущее соответствие с ранее сохранённым.

### Сложность

- Временная сложность: `O(n)`, где `n` - длина строк.
- Дополнительная память: `O(k)`, где `k` - количество уникальных символов. В худшем случае `O(n)`.

## Задача 2. Поиск пропущенного числа

### Условие

Дана последовательность натуральных чисел от 1 до n, в которой отсутствует ровно одно число. Нужно найти это число.

Пример:

```python
nums = [1, 2, 3, 4, 5, 6, 8, 9, 10, 11]
print(missing_number(nums))
# 7
```

### Решение

```python
def missing_number(nums):
    n = len(nums) + 1
    expected_sum = n * (n + 1) // 2
    actual_sum = sum(nums)
    return expected_sum - actual_sum


nums = [1, 2, 3, 4, 5, 6, 8, 9, 10, 11]
print(missing_number(nums))
```

### Логика решения

Если в последовательности от 1 до n отсутствует одно число, то длина списка будет `n - 1`. Поэтому настоящее значение `n` можно найти как:

```python
n = len(nums) + 1
```

Затем вычисляется ожидаемая сумма последовательности от 1 до n по формуле:

```python
n * (n + 1) // 2
```

После этого считается фактическая сумма элементов списка. Разница между ожидаемой и фактической суммой даёт пропущенное число.

### Сложность

- Временная сложность: `O(n)`, потому что требуется один проход по списку для вычисления суммы.
- Дополнительная память: `O(1)`, потому что используются только несколько вспомогательных переменных.

## Задача 3. Разложение числа на простые множители

### Условие

Нужно разложить натуральное число на простые множители.

Пример:

```python
n = 56
print(prime_factors(n))
# [2, 2, 2, 7]
```

### Решение

```python
def prime_factors(n):
    factor_list = []
    divider = 2

    while n > 1:
        if n % divider == 0:
            factor_list.append(divider)
            n = n // divider
        else:
            divider += 1

    return factor_list


n = 2312317858
print(prime_factors(n))
```

### Логика решения

Алгоритм последовательно проверяет делители, начиная с 2. Если число делится на текущий делитель без остатка, делитель добавляется в список простых множителей, а само число уменьшается через деление.

Если число на текущий делитель не делится, делитель увеличивается на 1.

Процесс продолжается до тех пор, пока `n` не станет равным 1.

### Сложность

- Временная сложность: `O(n)` в худшем случае для данной реализации.
- Дополнительная память: `O(log n)`, так как память расходуется на хранение найденных простых множителей.

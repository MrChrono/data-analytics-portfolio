# SQL Practice

## Описание проекта

В этой папке собраны учебные SQL-задачи и практические запросы, которые демонстрируют базовые навыки работы с реляционными данными.

Проект это часть портфолио Junior Data Analyst: демонстрирует умение читать условие задачи, понимать структуру таблиц, формулировать логику выборки, писать SQL-запросы и объяснять результат.

## Основной фокус

- выборка данных через `SELECT`;
- фильтрация через `WHERE`;
- сортировка через `ORDER BY`;
- группировка и агрегация через `GROUP BY`;
- фильтрация агрегированных данных через `HAVING`;
- соединение таблиц через `JOIN`;
- подзапросы;
- оконные функции;
- работа с датами;
- проверка логики запроса и граничных случаев.

sql-practice/

## Разделы практики

| Раздел | Что показывает |
|---|---|
| `basic_queries.sql` | Базовые SELECT-запросы, фильтры, сортировка, условия |
| `joins.sql` | Соединение таблиц, INNER JOIN, LEFT JOIN, FULL JOIN, проверка потерь данных |
| `aggregation.sql` | Группировки, суммы, средние, COUNT, HAVING |
| `window_functions.sql` | RANK, DENSE_RANK, ROW_NUMBER, расчёты по группам |
| `business_cases.sql` | Прикладные задачи: покупки, клиенты, рейтинги, активность, метрики |

## Формат оформления задач

Каждая задача будет оформляться в едином формате:

```sql
-- Задача:
-- Краткое описание бизнес-условия.

-- Таблицы:
-- table_1(column_1, column_2, ...)
-- table_2(column_1, column_2, ...)

-- Решение:
SELECT ...
FROM ...
WHERE ...;

-- Пояснение:
-- Почему запрос написан именно так и что он возвращает.
```

## Примеры задач

В проект могут входить задачи на темы:

- рейтинг абитуриентов по баллам;
- покупки клиентов за последний месяц;
- расчёт суммы заказов по пользователям;
- поиск клиентов ниже заданного порога активности;
- анализ фильмов и рейтингов;
- сравнение разных типов JOIN;
- ранжирование объектов с помощью оконных функций;
- поиск дублей и проверка качества данных.

## Что демонстрирует проект

Проект показывает навыки:

- понимания структуры данных;
- перевода бизнес-условия в SQL-запрос;
- работы с несколькими таблицами;
- расчёта агрегированных показателей;
- использования оконных функций;
- проверки логики запроса;
- аккуратного документирования решения.

# TopFace Media Test Assignment: Prolongation Coefficient Analysis

## Описание проекта

Проект выполнен в рамках тестового задания TopFace Media.

Цель работы — подготовить данные по клиентским проектам, рассчитать коэффициенты пролонгации за 2023 год и оформить результат в виде итогового аналитического отчёта.

Проект показывает полный цикл аналитической работы: проверка качества данных, очистка таблиц, формализация бизнес-логики, расчёт метрик, контроль корректности и подготовка отчётных таблиц для интерпретации результата.

## Бизнес-задача

Нужно рассчитать показатели пролонгации клиентских проектов:

- по всему отделу;
- по каждому аккаунт-менеджеру;
- по месяцам;
- за 2023 год.

Коэффициенты рассчитываются по суммам отгрузки, а не по количеству проектов. Поэтому значение коэффициента может быть выше 100%, если сумма отгрузки после пролонгации оказалась больше, чем сумма в последний месяц реализации проекта.

## Исходные данные

В проекте используются два исходных CSV-файла:

- `prolongations.csv` — информация о проектах, последнем месяце реализации и аккаунт-менеджере;
- `financial_data.csv` — финансовые данные по проектам и месяцам.

## Основные бизнес-правила расчёта

В расчётах использована следующая логика:

- `month` в `prolongations.csv` считается последним месяцем реализации проекта;
- аккаунт-менеджер берётся из `prolongations.csv`, так как этот файл является основным источником по условию задачи;
- коэффициент 1-го месяца считается как отношение отгрузки в первый месяц после завершения проекта к отгрузке в последний месяц реализации;
- коэффициент 2-го месяца считается только для проектов, которые не были пролонгированы в первый месяц;
- специальные значения `стоп` и `end` исключают соответствующие записи из расчёта, если встречаются в последний месяц реализации или ранее;
- значение `в ноль` обрабатывается как специальный случай: если в расчётном месяце отгрузка равна нулю, для расчёта используется отгрузка предыдущего месяца;
- годовой коэффициент считается как сумма числителей, делённая на сумму знаменателей, а не как среднее месячных коэффициентов.

## Что было сделано

### 1. Первичная проверка данных

- проверена структура исходных таблиц;
- проверено покрытие `id` между `prolongations.csv` и `financial_data.csv`;
- проанализированы аккаунт-менеджеры в двух источниках;
- найдены и обработаны дубликаты;
- проверены специальные значения `в ноль`, `стоп`, `end`;
- проверены неожиданные нечисловые значения в финансовых колонках.

### 2. Очистка и подготовка данных

- русские текстовые месяцы преобразованы в формат даты;
- удалены точные дубликаты;
- обработаны конфликтные записи;
- исключены записи, которые не должны участвовать в расчёте из-за `стоп/end`;
- финансовая таблица преобразована из широкого формата в длинный;
- значения отгрузки очищены и приведены к числовому формату.

### 3. Расчёт метрик

- сформирована расчётная база пролонгаций;
- рассчитана отгрузка последнего месяца реализации проекта;
- рассчитана отгрузка в первый месяц после завершения;
- рассчитана отгрузка во второй месяц после завершения;
- добавлены флаги пролонгации в 1-й и 2-й месяц;
- рассчитаны коэффициенты по отделу, менеджерам и месяцам.

### 4. Контроль корректности

В проекте добавлены проверки:

- отсутствие дублей в расчётной базе;
- отсутствие конфликтов по аккаунт-менеджерам;
- отсутствие пропусков в расчётных отгрузках;
- отсутствие нулевых и отрицательных знаменателей;
- корректность исключения проектов без достаточных финансовых данных;
- дополнительная проверка причин низкого коэффициента 2-го месяца.

## Результаты анализа

По итогам 2023 года:

- коэффициент пролонгации по отделу в первый месяц составил **53,61%**;
- из **313** попыток пролонгации положительная отгрузка в первый месяц была зафиксирована в **158** случаях;
- коэффициент пролонгации через месяц составил **5,11%**;
- из **152** попыток пролонгации во второй месяц положительная отгрузка появилась только в **15** случаях;
- основной объём пролонгаций формируется в первый месяц после завершения проекта;
- если проект не продлевается сразу, вероятность его последующего возврата заметно снижается.

По месячной динамике лучший результат отдела по коэффициенту 1-го месяца наблюдался:

- в октябре — **81,77%**;
- в феврале — **78,26%**;
- в марте — **64,60%**.

Самый слабый месяц по первичной пролонгации — июнь: **22,10%**.

Среди менеджеров по годовому коэффициенту 1-го месяца высокие результаты показали:

- Смирнова Ольга Владимировна — **68,35%**;
- Михайлов Андрей Сергеевич — **66,49%**;
- Соколова Анастасия Викторовна — **57,91%**.

Отдельно важно учитывать базу расчёта: у менеджеров с небольшим количеством проектов коэффициент может выглядеть высоким, но быть менее устойчивым для управленческих выводов.

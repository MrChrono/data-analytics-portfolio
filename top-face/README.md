# TopFace Media Test Assignment: Prolongation Coefficient Analysis

## Описание проекта

Проект выполнен в рамках тестового задания TopFace Media.  
Цель работы — подготовить данные, рассчитать коэффициенты пролонгации клиентских проектов за 2023 год и сформировать итоговые таблицы для отчёта в Google Sheets.

Проект демонстрирует полный цикл аналитической работы: от проверки качества данных и формализации бизнес-логики до расчёта метрик, контроля корректности и подготовки отчётных CSV-файлов.

## Бизнес-задача

Необходимо рассчитать показатели пролонгации проектов:

- по всему отделу;
- по каждому аккаунт-менеджеру;
- по месяцам;
- за 2023 год.

Коэффициенты рассчитываются по суммам отгрузки, а не по количеству проектов.

## Исходные данные

В проекте используются два CSV-файла:

- `prolongations.csv` — информация о проектах, последнем месяце реализации и аккаунт-менеджере;
- `financial_data.csv` — финансовые данные по проектам и месяцам.

## Основные бизнес-правила

В расчётах использованы следующие правила:

- `month` в `prolongations.csv` — последний месяц реализации проекта;
- аккаунт-менеджер берётся из `prolongations.csv`, так как это основной источник по условию задачи;
- коэффициент 1-го месяца считается как отношение отгрузки в первый месяц после завершения проекта к отгрузке в последний месяц реализации;
- коэффициент 2-го месяца считается только для проектов, которые не были пролонгированы в 1-й месяц;
- значения `стоп` и `end` исключают соответствующую запись из расчёта, если встречаются в последний месяц реализации или ранее;
- значение `в ноль` обрабатывается как специальный случай: если отгрузка в месяце равна нулю, для расчёта используется отгрузка предыдущего месяца;
- годовой коэффициент считается как сумма числителей, делённая на сумму знаменателей, а не как среднее месячных коэффициентов.

## Что было сделано

### 1. Первичная проверка данных

- проверена структура исходных таблиц;
- проверено покрытие `id` между `prolongations.csv` и `financial_data.csv`;
- проанализированы справочники аккаунт-менеджеров;
- найдены и обработаны дубликаты;
- проверены специальные значения: `в ноль`, `стоп`, `end`;
- проверены неожиданные нечисловые значения в финансовых колонках.

### 2. Очистка и подготовка данных

- преобразованы русские текстовые месяцы в формат `datetime`;
- удалены точные дубликаты в `prolongations`;
- обработаны конфликтные записи;
- исключены записи, которые не должны участвовать в расчёте из-за `стоп/end`;
- финансовые данные преобразованы из широкого формата в длинный;
- значения отгрузки очищены и приведены к числовому формату.

### 3. Расчётная логика

- сформирована база пролонгаций;
- рассчитана отгрузка последнего месяца реализации проекта;
- рассчитана отгрузка в первый и второй месяцы после завершения;
- добавлены флаги пролонгации в 1-й и 2-й месяц;
- сформированы базы для расчёта коэффициентов за 2023 год.

### 4. Проверка корректности

В проекте добавлены контрольные проверки:

- отсутствие дублей в расчётной базе;
- отсутствие конфликтов по аккаунт-менеджерам;
- отсутствие пропусков в расчётных отгрузках;
- отсутствие нулевых и отрицательных знаменателей;
- корректность исключения проектов без достаточных финансовых данных;
- дополнительная проверка причин низкого коэффициента 2-го месяца.

## Результаты

По итогам анализа за 2023 год:

- годовой коэффициент пролонгации отдела в 1-й месяц составил около **53,6%**;
- годовой коэффициент пролонгации во 2-й месяц составил около **5,1%**;
- основной объём пролонгаций формируется в первый месяц после завершения проекта;
- низкий коэффициент 2-го месяца объясняется структурой данных: во второй месяц попадают только проекты, которые не были пролонгированы сразу;
- коэффициенты выше 100% не считаются ошибкой, так как метрика рассчитывается по сумме отгрузки, а сумма после пролонгации может быть выше, чем в последний месяц реализации.
